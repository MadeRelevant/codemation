# Define Workflows

Codemation workflows are plain TypeScript modules that export a workflow definition.

In a generated app, `codemation.config.ts` registers workflow discovery like this:

```ts
register(app: CodemationAppContext) {
  app.discoverWorkflows("src/workflows");
}
```

That means files under `src/workflows` are treated as workflow modules.

## The simplest workflow

The default starter template ships with a small example:

```ts
import { createWorkflowBuilder, ManualTrigger, MapData } from "@codemation/core-nodes";

type SeedJson = Readonly<{ step: string }>;

export default createWorkflowBuilder({ id: "wf.starter.hello", name: "Starter Hello" })
  .trigger(new ManualTrigger<SeedJson>("Start", [{ json: { step: "start" } }]))
  .then(
    new MapData<SeedJson, SeedJson & { hello: boolean }>("Hello", (item) => ({
      ...item.json,
      hello: true,
    })),
  )
  .build();
```

This example shows the typical builder flow:

1. Create a builder with an `id` and `name`
2. Start with a trigger
3. Add one or more nodes with `.then(...)`
4. Finish with `.build()`

## Where to put workflow files

Use folders that reflect your app, for example:

```text
src/workflows/
  starter/
    hello.ts
  integrations/
    crm-sync.ts
  ai/
    classify-inbound-email.ts
```

Codemation is happy with multiple workflow files and nested folders as long as they are under a discovered directory.

## Branching and control flow

The workflow builder supports more than just straight lines. Common patterns include:

- `If` for branching
- `.when({ true: [...], false: [...] })` for branch-specific paths
- `Wait` for pauses or delayed continuation
- `SubWorkflow` for composing larger flows from smaller ones

## Real-world example shape

The repo also contains richer examples, such as a Gmail trigger that:

1. receives incoming mail,
2. reads attachments,
3. uses credentials to talk to Azure services,
4. runs an AI classification step,
5. branches based on the result.

That should give you a good mental model: workflows can stay readable even when they mix triggers, credentials, AI steps, and branching.

## Workflow authoring tips

- Give workflows stable IDs such as `wf.sales.rfq-triage`
- Keep workflow files small and focused
- Extract reusable helpers when prompts, mappings, or schemas grow
- Prefer one workflow file per business flow instead of giant all-in-one files
