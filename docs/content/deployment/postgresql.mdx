# Use PostgreSQL

Codemation does not start with SQLite.

The default local experience uses **embedded Postgres via PGlite**, which is perfect for quick local development because it avoids external database setup.

When you are ready for:

- shared environments,
- multiple processes,
- real production deployments,
- BullMQ workers,

switch to a normal TCP PostgreSQL database.

## When to switch

Stay on the local embedded setup when:

- you are developing alone,
- one process is enough,
- you want the shortest path to first success.

Switch to PostgreSQL when:

- you run web and worker processes separately,
- multiple instances need the same database,
- you want a shared staging or production environment.

## Environment setup

The starter template already hints at the required environment variables:

```bash
# DATABASE_URL=postgresql://user:password@127.0.0.1:5432/codemation
```

When you use PostgreSQL, define `DATABASE_URL` in `.env`.

## Config example

The template uses this runtime shape:

```ts
const useRedisRuntime = Boolean(process.env.REDIS_URL);
const databaseUrl = process.env.DATABASE_URL?.trim();

database: useRedisRuntime
  ? { kind: "postgresql" as const, url: databaseUrl! }
  : { kind: "pglite" as const, pgliteDataDir: ".codemation/pglite" };
```

For a dedicated PostgreSQL setup, the intent is the same even if you remove the conditional:

```ts
database: {
  kind: "postgresql" as const,
  url: process.env.DATABASE_URL ?? "",
}
```

## Run migrations

After setting `DATABASE_URL`, apply migrations:

```bash
pnpm exec codemation db migrate
```

## A useful mental model

- **PGlite**: great local default, single-process oriented
- **PostgreSQL**: shared runtime database for real deployments
