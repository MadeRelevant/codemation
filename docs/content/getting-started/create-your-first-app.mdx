# Create Your First App

The fastest way to start with Codemation is the published initializer:

```bash
npm create codemation@latest my-app
```

You can also use:

```bash
pnpm create codemation
yarn create codemation
```

## Prerequisites

- Node.js 20 or newer
- A package manager such as `npm`, `pnpm`, or `yarn`

## Choose a template

The initializer supports:

- `default`: the best starting point for most apps
- `minimal`: a smaller starter with fewer moving parts in the initial file layout

Example:

```bash
npm create codemation@latest my-app -- --template default
```

Useful options:

- `--template <id>`
- `--list-templates`
- `--force`
- `--non-interactive`
- `--yes`

## What gets generated

A generated app centers around a few files:

```text
my-app/
  codemation.config.ts
  .env
  src/
    branding/
    workflows/
      starter/
        hello.ts
```

The generated project already includes:

- the Codemation CLI,
- core workflow packages,
- a sample workflow,
- local auth defaults for development,
- an embedded Postgres-compatible local database via PGlite,
- a `pnpm dev` workflow that runs `codemation dev`.

## First successful local run

Inside the generated app:

```bash
pnpm install
pnpm exec codemation db migrate
pnpm exec codemation user create --email you@example.com --password 'your-secure-password'
pnpm dev
```

If you scaffolded interactively, the generator can already guide you through some of these steps.

## Local database default

The default template does **not** start with SQLite.

Instead, it uses **embedded Postgres via PGlite**, which gives you a zero-setup local database under `.codemation/pglite`. That is great for development and evaluation because you can get started without provisioning a separate PostgreSQL server.

## Generated config at a glance

The template `codemation.config.ts` does three important things immediately:

1. Loads `.env`
2. Discovers workflows from `src/workflows`
3. Switches between local embedded storage and a production-style queued setup based on environment variables

That means the starter app can stay simple locally, but still gives you a clean path toward PostgreSQL, Redis, BullMQ, and workers later.
